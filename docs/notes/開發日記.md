# 為何每次進入網站都超慢?

```
GET / 307 in 12985ms
   ○ Compiling /tasks ...
   ✓ Compiled /tasks in 2.3s (997 modules)
   GET /tasks 200 in 2814ms
```

從這段 log 看，主要是兩件事疊加：

  1. / 會先重導向到 /tasks

  // src/app/page.tsx
  redirect("/tasks");

  所以瀏覽器實際上會發生兩次請求：

  GET /       → 307
  GET /tasks  → 200

  2. 你目前跑的是開發模式，首次進入 /tasks 會即時編譯

  ○ Compiling /tasks ...
  ✓ Compiled /tasks in 2.3s

  Next.js next dev 的首次編譯、載入約 997 個 modules、Prisma 初始化，再加上資料庫查詢，都會讓第一次載入明
  顯變慢。之後重新整理通常會快很多。

  另外 /tasks 目前還會重複驗證登入狀態：

  - src/app/(app)/layout.tsx 呼叫一次 auth()
  - src/app/(app)/tasks/page.tsx 又呼叫一次 auth()
  - getLists()、getTags()、getTodayViewGrouped() 內部的 requireAuth() 又各自呼叫一次 auth()

  而且頁面會同時執行：

  - 1 次 lists 查詢
  - 1 次 tags 查詢
  - 2 次 tasks 查詢（逾期、今日）

  因此開發環境看到 2～13 秒並不罕見，但若每次都這麼慢，建議先確認：

  npm run build
  npm start

  再測試正式模式。如果正式模式明顯變快，問題主要是開發模式編譯成本。

  目前最值得改善的是：

  - /tasks 頁面不要重複呼叫 auth()
  - 將已取得的 userId 傳入查詢函式，避免每個 Server Action 再驗證一次
  - 為 Task 增加符合查詢條件的複合索引，例如 userId/status/dueDate
  - 首頁可直接使用 /tasks 作為入口，減少一次重導向

  所以這段 log 中，最直接的原因是「開發模式首次編譯 + / 額外重導向」；資料查詢與重複認證則是後續可優化的
  部分。
