## 修改的需求

### 需求：任務實體結構
任務實體應包含用於管理任務詳細資訊的欄位。任務記錄由使用者擁有，可以屬於清單。

**已更新為包括時間管理欄位：**
- `id`：唯一識別碼
- `userId`：擁有者使用者 ID
- `listId`：選擇性的清單關聯
- `title`：任務名稱（必填）
- `note`：選擇性說明
- `dueDate`：選擇性截止日期（DateTime）
- `dueTime`：選擇性任務開始時間（HH:MM 格式）— **新增**
- `endTime`：選擇性任務結束時間（HH:MM 格式）— **新增**
- `allDay`：全天狀態的布林旗標（預設：false）— **新增**
- `priority`：優先級（高、中、低、無）
- `status`：任務狀態（待做、完成）
- `completedAt`：標記完成時的時間戳
- `sortOrder`：手動排序順序
- `createdAt`：記錄建立時間戳
- `updatedAt`：記錄更新時間戳
- `tags`：與標籤的多對多關係
- `subtasks`：與子任務的一對多關係

#### 情景：使用時間欄位建立任務
- **當** 建立新任務
- **那麼** dueTime 和 endTime 可以選擇性設定
- **並且** allDay 預設為 false

#### 情景：更新任務時間資訊
- **當** 更新現有任務
- **那麼** dueTime、endTime 和 allDay 可以獨立修改
- **並且** 驗證確保 endTime >= dueTime

#### 情景：按日期和時間查詢任務
- **當** 擷取日曆視圖的任務
- **那麼** 任務可以按 dueDate 篩選
- **並且** 任務可以按 dueTime 排序
- **並且** 可以區分帶/不帶時間的任務

#### 情景：保留向後相容性
- **當** 現有任務有 dueDate，但沒有 dueTime/endTime/allDay
- **那麼** 它們被視為無時間任務（僅顯示，沒有時間定位）
- **並且** 現有任務的 allDay 預設為 false
